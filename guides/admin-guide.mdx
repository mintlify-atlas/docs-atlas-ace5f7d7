# Admin Guide

Welcome to the ArrendaOco administrator guide. This comprehensive resource covers all platform management capabilities available to system administrators.

## Admin Access & Permissions

### Becoming an Administrator

Admin access is granted through:
- Database-level assignment of `es_admin = true` on usuario record
- Assignment of the "admin" role in the role system

Users with either flag have full administrative capabilities.

### Admin Middleware Protection

All admin routes are protected by the `admin` middleware:
- Verifies user is authenticated
- Checks for `es_admin` flag or "admin" role
- Returns 403 Forbidden if unauthorized
- Prevents accidental access by regular users

## User Management

### Viewing All Users

The user management dashboard (`/admin/usuarios`) provides:

- **Complete User List**: All registered usuarios with pagination (10 per page)
- **User Details**:
  - Full name
  - Email address
  - Assigned roles (inquilino, propietario, admin)
  - Account status (activo/inactivo)
  - Registration date
- **Search Functionality**: Filter users by:
  - Name (partial match)
  - Email address (partial match)
- **Quick Actions**: Edit, view, or delete buttons for each user

### Creating New Users

Manually create user accounts:

<Steps>
  <Step title="Navigate to User Creation">
    Go to `/admin/usuarios/create` or click "Create User" button
  </Step>
  
  <Step title="Enter Required Information">
    Provide:
    - **Full Name** (nombre): Minimum 3 characters, maximum 255
    - **Email**: Valid, unique email address
    - **Password**: Minimum 8 characters
    - **Password Confirmation**: Must match
    - **Status**: Select "activo" or "inactivo" (defaults to activo)
  </Step>
  
  <Step title="Submit Form">
    Click create - password is automatically hashed with BCrypt
  </Step>
  
  <Step title="Assign Roles">
    After creation, edit user to assign specific roles (inquilino, propietario, admin)
  </Step>
</Steps>

#### Validation Rules

- **Name**: Required, 3-255 characters
- **Email**: Required, valid format, unique in database
- **Password**: Required, minimum 8 characters, must be confirmed
- **Status**: Must be "activo" or "inactivo"

<Warning>
Manually created users receive default tenant role. Remember to assign additional roles (landlord, admin) as needed through the edit interface.
</Warning>

</Tip>

### Managing User Status

#### Active Users (activo)

- Can log into the platform
- Access all features based on assigned roles
- Publish properties if landlord
- Create contracts and payments

#### Inactive Users (inactivo)

- **Cannot log in** - authentication fails
- **Cannot perform operations** - all functionality restricted
- **Data preserved** - properties, contracts, reviews remain
- **Reversible** - can be reactivated at any time

### Deleting Users

Permanently remove user accounts:

<Steps>
  <Step title="Initiate Deletion">
    Click delete button on user in the admin users list
  </Step>
  
  <Step title="System Verification Checks">
    Automatic validation prevents deletion if:
    - User is the currently logged-in admin (cannot self-delete)
    - User has active contracts as landlord
    - User has active contracts as tenant
    - User has published properties
  </Step>
  
  <Step title="Confirm Deletion">
    If all checks pass, confirm the permanent deletion
  </Step>
  
  <Step title="User Removed">
    Account and associated authentication data permanently deleted
  </Step>
</Steps>

#### Deletion Safety Checks

The system prevents deletion when:

```php
// Prevents admin self-deletion
if ($id == auth()->id()) {
    return error('Cannot delete your own admin account');
}

// Checks for active contracts
if ($usuario->contratosComoPropietario()->where('estatus', 'activo')->exists() ||
    $usuario->contratosComoInquilino()->where('estatus', 'activo')->exists()) {
    return error('User has active contracts');
}

// Checks for published properties
if ($usuario->inmuebles()->exists()) {
    return error('User has registered properties');
}
```

<Warning>
Deletion is permanent and irreversible. Always use status deactivation (inactivo) for temporary restrictions. Only delete accounts for spam, duplicates, or when explicitly requested after all obligations are cleared.
</Warning>

### Role Management

Administrators can assign multiple roles to users:

#### Available Roles

- **Inquilino** (Tenant): Default role for all users
  - Search and view properties
  - Manage favorites
  - Create reviews
  - Access rental contracts as tenant

- **Propietario** (Landlord): Property owners
  - All tenant capabilities
  - Publish properties
  - Manage property listings
  - Create rental contracts
  - Track payments

- **Admin** (Administrator): Platform managers
  - All system access
  - User management
  - Content moderation
  - Report generation
  - Override permissions

#### Assigning Roles

<Steps>
  <Step title="Edit User">
    Access the user edit form
  </Step>
  
  <Step title="Select Roles">
    Choose from available role checkboxes or dropdown
  </Step>
  
  <Step title="Save Changes">
    Roles take effect immediately
  </Step>
</Steps>

<Note>
Users can have multiple roles simultaneously. A user can be both a tenant and landlord, allowing them to search for properties while also managing their own listings.
</Note>

  </Step>

    - Keep records of dispute details
    - Note actions taken
    - Track outcomes
    - Identify patterns for policy improvement

## Troubleshooting Admin Issues

### Can't Access Admin Routes

**Problem**: 403 Forbidden error on admin pages

**Solutions**:
- Verify your account has `es_admin = true` in database
- Check if you have the "admin" role assigned
- Ensure you're logged in with admin account
- Clear cache and retry
- Contact technical team if issue persists

### Can't Delete User

**Problem**: Deletion fails with error message

**Solutions**:
- Check error message for specific reason
- Verify user has no active contracts
- Ensure user has no published properties
- Confirm you're not trying to delete yourself
- Have landlord transfer properties first if needed

### Report Generation Fails

**Problem**: PDF reports don't download

**Solutions**:
- Check server PDF generation library (DomPDF) is installed
- Verify sufficient server memory for report generation
- Test with smaller data sets
- Check error logs for specific failure reason
- Contact technical team for server configuration

### Property Edit Doesn't Save

**Problem**: Changes to properties don't persist

**Solutions**:
- Verify validation passes (check error messages)
- Ensure file uploads don't exceed size limits
- Check for conflicts with active contracts
- Verify database connection
- Review error logs for specific issues

## Platform Policies & Guidelines

### Content Standards

Enforce these platform standards:

#### Property Listings
- **Accurate Information**: Descriptions must match actual property
- **Real Photos**: Images must show the actual property being listed
- **Complete Details**: All required fields should be filled
- **Appropriate Pricing**: Prices should reflect market rates
- **Location Accuracy**: Coordinates and addresses must be correct

#### User Reviews
- **Authentic Experience**: Reviews from actual rental contracts only
- **Respectful Language**: No profanity or personal attacks
- **Relevant Content**: Reviews about rental experience, not unrelated topics
- **Honest Feedback**: Both positive and negative reviews allowed
- **No Spam**: Prohibit promotional or repeated content

### User Conduct

Expected behavior for all users:

- **Respectful Communication**: Professional interactions between landlords and tenants
- **Truthfulness**: Accurate information in listings, profiles, and communications
- **Privacy Respect**: Don't share others' personal information
- **Legal Compliance**: Follow local Ocosingo and Mexican rental laws
- **Platform Use**: Use ArrendaOco for legitimate rental purposes only

- **Capabilities**: 
  - Property recommendations
  - Platform assistance
  - General rental information
- **Limitations**: AI responses based on database context
- **Customization**: Users can personalize assistant name

<Note>
Monitor Arrendito usage for inappropriate responses or technical issues. Coordinate with technical team if AI provides incorrect information.
</Note>

### Payment System

Understand the payment tracking workflow:

1. **Contract Creation**: Generates monthly payment schedule
2. **Payment Records**: Each month has pending payment entry
3. **Payment Processing**: Landlord marks as paid with reference
4. **Late Fees**: Automatic calculation for overdue payments
5. **Statements**: PDF generation for payment history

Admins may need to:
- Verify payment status in disputes
- Correct erroneous payment records
- Review late fee calculations
- Access financial statements for conflict resolution

### Social Authentication

Platform supports:
- **Google OAuth**: Users can register/login with Google accounts
- **Facebook** (may be disabled): Alternative social login

Admin considerations:
- Social login users may not have traditional passwords
- Google ID and Facebook ID stored separately from email/password
- Social users still subject to same policies and management
