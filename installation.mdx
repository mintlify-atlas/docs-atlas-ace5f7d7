## Overview

This comprehensive guide covers installing ArrendaOco for both development and production environments. For a quick local setup, see the [Quickstart Guide](/quickstart).

## System Requirements

### Minimum Requirements

<CardGroup cols={2}>
  <Card title="PHP" icon="php">

    **Required Extensions**:
    - BCMath
    - Ctype
    - Fileinfo
    - JSON
    - Mbstring
    - OpenSSL
    - PDO
    - Tokenizer
    - XML
    - cURL
    - GD or Imagick
  </Card>
  
  <Card title="Database" icon="database">
    **MySQL**: 8.0+ (recommended)
    
    **PostgreSQL**: 13+ (supported)
    
    **SQLite**: 3.35+ (development only)
    
    **Character Set**: UTF8MB4
  </Card>
  
  <Card title="Node.js" icon="node-js">
    **Version**: 18.0 or higher
    
    **Package Manager**: npm 9.0+
    
    **Required for**: Asset compilation (Vite)
  </Card>
  
  <Card title="Composer" icon="box">
    **Version**: 2.x
    
    **Required for**: PHP dependency management
  </Card>
</CardGroup>

### Server Requirements (Production)

- **Web Server**: Apache 2.4+ or Nginx 1.18+
- **RAM**: Minimum 512MB, 2GB recommended
- **Storage**: 1GB+ for application files and uploads
- **SSL Certificate**: Required for HTTPS (Let's Encrypt recommended)

### Verify PHP Installation

Check your PHP version and extensions:

```bash
php -v
php -m | grep -E "pdo|mbstring|openssl|tokenizer|xml|curl|gd"
```

    </Steps>

    ArrendaOco includes a Composer script for automated setup:
    
    ```bash

    ```
    
    This script automatically:
    1. Installs Composer dependencies
    2. Copies `.env.example` to `.env`
    3. Generates application key
    4. Runs database migrations
    5. Installs npm packages
    6. Builds production assets
    
    <Warning>

    ```bash
    ./vendor/bin/sail npm install
    ./vendor/bin/sail npm run build
    ```

## Environment Configuration

### Core Application Settings

Edit your `.env` file with these essential configurations:

```bash .env
# Application
APP_NAME=ArrendaOco
APP_ENV=production  # Use 'local' for development
APP_KEY=base64:xxx  # Generated by key:generate
APP_DEBUG=false     # NEVER true in production
APP_URL=https://yourdomain.com
APP_TIMEZONE=America/Mexico_City
APP_LOCALE=es

# Database
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=bd_arrendaoco
DB_USERNAME=your_db_user
DB_PASSWORD=your_secure_password

# Cache & Sessions (Production)
CACHE_STORE=redis
SESSION_DRIVER=redis
QUEUE_CONNECTION=redis

# Redis
REDIS_HOST=127.0.0.1
REDIS_PASSWORD=null
REDIS_PORT=6379
```

<Warning>
  **Security Critical**: 
  - Set `APP_DEBUG=false` in production
  - Use strong database passwords
  - Never commit `.env` to version control
</Warning>

## Server Configuration

### Apache Configuration

Create a virtual host configuration:

```apache /etc/apache2/sites-available/arrendaoco.conf
<VirtualHost *:80>
    ServerName yourdomain.com
    ServerAlias www.yourdomain.com
    DocumentRoot /var/www/arrendaoco/public

    <Directory /var/www/arrendaoco/public>
        Options -Indexes +FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>

    ErrorLog ${APACHE_LOG_DIR}/arrendaoco-error.log
    CustomLog ${APACHE_LOG_DIR}/arrendaoco-access.log combined
</VirtualHost>
```

Enable the site and required modules:

```bash
sudo a2enmod rewrite
sudo a2ensite arrendaoco.conf
sudo systemctl reload apache2
```

### Nginx Configuration

Create a server block:

```nginx /etc/nginx/sites-available/arrendaoco
server {
    listen 80;
    listen [::]:80;
    server_name yourdomain.com www.yourdomain.com;
    root /var/www/arrendaoco/public;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    index index.php;

    charset utf-8;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location = /favicon.ico { access_log off; log_not_found off; }
    location = /robots.txt  { access_log off; log_not_found off; }

    error_page 404 /index.php;

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}
```

Enable the site:

```bash
sudo ln -s /etc/nginx/sites-available/arrendaoco /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### SSL Configuration (Let's Encrypt)

Install and configure SSL certificate:

```bash
# Install Certbot
sudo apt install certbot python3-certbot-apache  # For Apache
# OR
sudo apt install certbot python3-certbot-nginx   # For Nginx

# Obtain certificate
sudo certbot --apache -d yourdomain.com -d www.yourdomain.com
# OR
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com

# Update .env
APP_URL=https://yourdomain.com
```

<Note>
  SSL certificates auto-renew. Test renewal with: `sudo certbot renew --dry-run`
</Note>

### File Permissions

Set correct ownership and permissions:

```bash
# Set owner (replace 'www-data' with your web server user)
sudo chown -R www-data:www-data /var/www/arrendaoco

# Set directory permissions
sudo find /var/www/arrendaoco -type d -exec chmod 755 {} \;

# Set file permissions
sudo find /var/www/arrendaoco -type f -exec chmod 644 {} \;

# Storage and cache directories need write access
sudo chmod -R 775 /var/www/arrendaoco/storage
sudo chmod -R 775 /var/www/arrendaoco/bootstrap/cache
```

## Database Migration

### Run Migrations

Create all database tables:

```bash
php artisan migrate --force
```

The `--force` flag is required in production (when `APP_ENV=production`).

</Warning>

### Queue Worker Setup

ArrendaOco uses queues for scheduled tasks. Set up a supervisor process:

```ini /etc/supervisor/conf.d/arrendaoco-worker.conf
[program:arrendaoco-worker]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/arrendaoco/artisan queue:work redis --sleep=3 --tries=3 --max-time=3600
autostart=true
autorestart=true
stopasgroup=true
killasgroup=true
user=www-data
numprocs=2
redirect_stderr=true
stdout_logfile=/var/www/arrendaoco/storage/logs/worker.log
stopwaitsecs=3600
```

Activate the worker:

```bash
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start arrendaoco-worker:*
```

### Scheduled Tasks (Cron)

ArrendaOco has automated commands for payment processing. Add to crontab:

```bash
crontab -e
```

Add this line:

```cron
* * * * * cd /var/www/arrendaoco && php artisan schedule:run >> /dev/null 2>&1
```

Key scheduled commands:
- `app:calcular-recargos` - Apply late fees to overdue payments
- `app:marcar-pagos-vencidos` - Mark payments as overdue
- `app:generar-estados-cuenta-mensuales` - Generate monthly statements

See `app/Console/Commands/` for implementations.

## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="500 Internal Server Error" icon="circle-exclamation">
    **Causes**:
    - Missing `.env` file
    - Wrong file permissions
    - PHP errors
    
    **Solutions**:
    ```bash
    # Check logs
    tail -f storage/logs/laravel.log
    
    # Fix permissions
    chmod -R 775 storage bootstrap/cache
    
    # Clear cache
    php artisan config:clear
    php artisan cache:clear
    ```
  </Accordion>
  
  <Accordion title="Database Connection Failed" icon="database">
    **Solutions**:
    ```bash
    # Test database connection
    php artisan tinker
    >>> DB::connection()->getPdo();
    
    # Verify .env settings
    cat .env | grep DB_
    
    # Check MySQL is running
    sudo systemctl status mysql
    ```
  </Accordion>
  
  <Accordion title="Assets Not Loading" icon="image">
    **Solutions**:
    ```bash
    # Rebuild assets
    npm run build
    
    # Fix storage link
    php artisan storage:link
    
    # Check permissions
    ls -la public/storage
    ```
  </Accordion>
  
  <Accordion title="Queue Jobs Not Processing" icon="list-check">
    **Solutions**:
    ```bash
    # Check queue worker
    sudo supervisorctl status arrendaoco-worker:*
    
    # Restart workers
    php artisan queue:restart
    
    # Monitor queue
    php artisan queue:listen --verbose
    ```
  </Accordion>
  
  <Accordion title="Email Not Sending" icon="envelope">
    **Solutions**:
    ```bash
    # Test email config
    php artisan tinker
    >>> Mail::raw('Test', function($msg) { $msg->to('test@example.com')->subject('Test'); });
    
    # Check mail logs
    tail -f storage/logs/laravel.log | grep -i mail
    
    # Verify SMTP settings in .env
    ```
  </Accordion>
</AccordionGroup>

## Backup Strategy

### Database Backup

```bash
# Create backup script
cat > /usr/local/bin/backup-arrendaoco.sh << 'EOF'
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/backups/arrendaoco"
mkdir -p $BACKUP_DIR

# Database backup
mysqldump -u arrendaoco_user -p'password' bd_arrendaoco | gzip > $BACKUP_DIR/db_$DATE.sql.gz

# Files backup
tar -czf $BACKUP_DIR/files_$DATE.tar.gz /var/www/arrendaoco/storage/app/public

# Keep only last 30 days
find $BACKUP_DIR -name "*.gz" -mtime +30 -delete
EOF

chmod +x /usr/local/bin/backup-arrendaoco.sh
```

Add to crontab for daily backups at 2 AM:

```cron
0 2 * * * /usr/local/bin/backup-arrendaoco.sh
```

