## Overview

ArrendaOco uses environment variables to manage configuration across different environments. All configuration is centralized in the `.env` file at the root of your project.

<Warning>
  Never commit your `.env` file to version control. It contains sensitive credentials and API keys.
</Warning>

## Creating Your Environment File

When setting up ArrendaOco for the first time, copy the example file:

```bash
cp .env.example .env
php artisan key:generate
```

The `key:generate` command creates a secure `APP_KEY` used for encryption.

## Core Application Settings

### Application Identity

```env
APP_NAME="ArrendaOco"
APP_ENV=local
APP_DEBUG=true
APP_URL=http://localhost:8000
APP_TIMEZONE=UTC
APP_LOCALE=en
APP_FALLBACK_LOCALE=en
APP_FAKER_LOCALE=en_US
```

| Variable | Description | Default |
|----------|-------------|----------|
| `APP_NAME` | Application name shown in emails and UI | `Laravel` |
| `APP_ENV` | Environment type: `local`, `staging`, `production` | `production` |
| `APP_DEBUG` | Enable detailed error messages | `false` |
| `APP_URL` | Full URL of your application | `http://localhost` |
| `APP_TIMEZONE` | Application timezone | `UTC` |
| `APP_LOCALE` | Default language code | `en` |

<Warning>
  **Production Safety**: Always set `APP_DEBUG=false` in production. Debug mode exposes sensitive information like database credentials and file paths in error messages.
</Warning>

### Encryption Key

```env
APP_KEY=base64:your-generated-key-here
APP_CIPHER=AES-256-CBC
```

The `APP_KEY` is automatically generated by `php artisan key:generate`. This key encrypts:
- User sessions
- Cookies
- Password reset tokens
- Any data encrypted using Laravel's `encrypt()` helper

<Warning>
  Changing `APP_KEY` after deploying will invalidate all existing sessions and encrypted data. Users will be logged out and password reset tokens will become invalid.
</Warning>

## API Keys & Third-Party Services

### Gemini AI (ROCO Assistant)

ArrendaOco uses Google's Gemini AI for the intelligent ROCO chatbot assistant.

```env
GEMINI_API_KEY=your_gemini_api_key_here
GEMINI_MODEL=gemini-2.5-flash
```

| Model | Use Case | Status |
|-------|----------|--------|
| `gemini-2.5-flash` | Fast, cost-effective, adaptive | **Default (Recommended)** |
| `gemini-2.5-pro` | Complex reasoning tasks | For advanced use cases |
| `gemini-3-flash-preview` | Latest generation | Experimental |

**Getting your API key:**
1. Visit [Google AI Studio](https://makersuite.google.com/app/apikey)
2. Create or select a project
3. Generate an API key
4. Add it to your `.env` file

**Configuration location:** `config/services.php:50-53`

<Info>
  The Gemini AI key is used by `ArrenditoChatController` and `Api/ArrenditoController`. Without it, the ROCO chatbot will not function.
</Info>

### Social Login (OAuth)

ArrendaOco supports login via Google and Facebook using Laravel Socialite.

#### Google OAuth

```env
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret
GOOGLE_REDIRECT_URI=http://127.0.0.1:8000/auth/google/callback
```

**Setup:**
1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create a new project or select existing
3. Enable "Google+ API"
4. Create OAuth 2.0 credentials
5. Add authorized redirect URI: `http://127.0.0.1:8000/auth/google/callback`

#### Facebook OAuth

```env
FACEBOOK_CLIENT_ID=your_facebook_app_id
FACEBOOK_CLIENT_SECRET=your_facebook_app_secret
FACEBOOK_REDIRECT_URI=http://127.0.0.1:8000/auth/facebook/callback
```

**Setup:**
1. Go to [Facebook Developers](https://developers.facebook.com/)
2. Create a new app
3. Add "Facebook Login" product
4. Configure OAuth redirect URIs

**Configuration location:** `config/services.php:38-48`

<Warning>
  For local development, use `http://127.0.0.1:8000` instead of `http://localhost:8000`. Some OAuth providers treat these as different domains.
</Warning>

## Cache & Session

### Cache Configuration

```env
CACHE_STORE=database
CACHE_PREFIX=arrendaoco-cache-
```

Default cache store is `database`. Options: `file`, `redis`, `memcached`, `array`

**Configuration location:** `config/cache.php:18`

### Session Configuration

```env
SESSION_DRIVER=database
SESSION_LIFETIME=120
SESSION_ENCRYPT=false
SESSION_PATH=/
SESSION_DOMAIN=null
SESSION_SECURE_COOKIE=false
SESSION_SAME_SITE=lax
```

## Queue Configuration

```env
QUEUE_CONNECTION=database
```

ArrendaOco uses database-based queues by default. Options: `sync`, `database`, `redis`, `sqs`

**Configuration location:** `config/queue.php:16`

To process queued jobs:
```bash
php artisan queue:listen --tries=1
```

## Development vs Production

### Development Environment

```env
APP_ENV=local
APP_DEBUG=true
MAIL_MAILER=log
CACHE_STORE=file
QUEUE_CONNECTION=sync
```

### Production Environment

```env
APP_ENV=production
APP_DEBUG=false
APP_URL=https://yourdomain.com
MAIL_MAILER=smtp
CACHE_STORE=redis
QUEUE_CONNECTION=redis
SESSION_SECURE_COOKIE=true
```

<Warning>
  **Production Checklist:**
  - Set `APP_DEBUG=false`
  - Use HTTPS (`APP_URL=https://...`)
  - Set `SESSION_SECURE_COOKIE=true`
  - Use production mail driver (SMTP, SES, etc.)
  - Configure proper cache driver (Redis recommended)
  - Set up queue workers with supervisor
  - Enable SSL for database connection if remote
</Warning>

## Security Best Practices

1. **Never commit `.env` to git** - Already in `.gitignore`
2. **Use strong, unique passwords** - Especially for database and admin accounts
3. **Rotate API keys regularly** - Especially after team member changes
4. **Use environment-specific keys** - Don't share production keys with development
5. **Enable HTTPS in production** - Set `SESSION_SECURE_COOKIE=true`
6. **Restrict database access** - Use firewall rules to limit access

## Troubleshooting

### "Application key not set"

```bash
php artisan key:generate
```

### Changes not taking effect

```bash
php artisan config:clear
php artisan cache:clear
```

### Social login not working

1. Verify OAuth credentials in `.env`
2. Check redirect URIs match exactly
3. Use `http://127.0.0.1:8000` not `http://localhost:8000`
4. Confirm APIs are enabled in provider console

### ROCO chatbot not responding

1. Verify `GEMINI_API_KEY` is set
2. Check API key is valid at [Google AI Studio](https://makersuite.google.com/app/apikey)
3. Review logs: `storage/logs/laravel.log`

## Environment Variables Reference

For a complete list of available environment variables, run:

```bash
php artisan config:show
```

Or inspect these configuration files:
- `config/app.php` - Application settings
- `config/database.php` - Database connections
- `config/services.php` - Third-party services
- `config/mail.php` - Email configuration
- `config/filesystems.php` - Storage drivers
