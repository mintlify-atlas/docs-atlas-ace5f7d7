## Endpoint

```
PUT /api/inmuebles/{id}
PATCH /api/inmuebles/{id}
```

<Info>
  **Authentication Required**
  
  This endpoint requires authentication. Only the property owner or an admin can update a property.
</Info>

## Description

Updates an existing property listing. Supports partial updates - you only need to send the fields you want to change.

## Authorization

The request is authorized if:
- The authenticated user is the property owner (`propietario_id` matches user ID), OR
- The authenticated user has admin privileges

**Authorization Policy:** `app/Policies/InmueblePolicy.php`

## Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the property to update
</ParamField>

## Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
  ```
  Bearer YOUR_ACCESS_TOKEN
  ```
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Use `multipart/form-data` when uploading images, otherwise `application/json`
</ParamField>

## Body Parameters

All fields are optional. Only include the fields you want to update.

<ParamField body="titulo" type="string" optional>
  Property title (max 255 characters)
</ParamField>

<ParamField body="descripcion" type="string" optional>
  Property description
</ParamField>

<ParamField body="direccion" type="string" optional>
  Street address
</ParamField>

<ParamField body="renta_mensual" type="number" optional>
  Monthly rent amount in MXN (must be >= 0)
</ParamField>

<ParamField body="deposito" type="number" optional>
  Security deposit amount in MXN (must be >= 0)
</ParamField>

<ParamField body="estatus" type="string" optional>
  Property status: `disponible`, `rentado`, or `inactivo`
</ParamField>

<ParamField body="habitaciones" type="integer" optional>
  Number of bedrooms (must be >= 0)
</ParamField>

<ParamField body="banos" type="integer" optional>
  Number of bathrooms (must be >= 0)
</ParamField>

<ParamField body="metros" type="number" optional>
  Property size in square meters (must be >= 0)
</ParamField>

<ParamField body="imagenes" type="array" optional>
  Array of new image files to upload
  - Accepted formats: jpg, jpeg, png, gif, svg, webp
  - Max file size: 5MB per image
  - New images are added to existing images (existing images are preserved)
</ParamField>

## Implementation Details

**Source:** `app/Http/Controllers/Api/InmuebleController.php:112`

### Authorization Check

The `authorize('update', $inmueble)` method checks:
- If user is property owner
- If user has admin role

### Partial Updates

All validation rules use the `sometimes` prefix, meaning they only apply if the field is present in the request.

### Image Upload Behavior

1. New images are **added** to existing images (existing images are NOT deleted)
2. If no main image exists (`imagen` field is null), the first new image becomes the main image
3. All images are stored in `storage/inmuebles/`
4. Each image creates a new record in `imagenes_inmuebles` table

### Allowed Status Values

The `estatus` field accepts only:
- `disponible`: Property is available for rent
- `rentado`: Property is currently rented
- `inactivo`: Property listing is inactive

## Notes

- Only the property owner or admin can update a property
- Partial updates are supported - send only the fields you want to change
- New images are added to existing images (not replaced)
- The `tipo` field cannot be updated after creation
- Location fields (`ciudad`, `estado`, `codigo_postal`) cannot be updated via this endpoint
