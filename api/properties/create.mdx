## Endpoint

```
POST /api/inmuebles
```

<Info>
  **Authentication Required**
  
  This endpoint requires a valid authentication token. The authenticated user must have property owner privileges.
</Info>

## Description

Creates a new property listing. The authenticated user is automatically set as the property owner. Properties are created in Ocosingo, Chiapas by default with status `disponible`.

## Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
  ```
  Bearer YOUR_ACCESS_TOKEN
  ```
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `multipart/form-data` when uploading images
</ParamField>

## Body Parameters

<ParamField body="titulo" type="string" required>
  Property title (max 255 characters)
</ParamField>

<ParamField body="descripcion" type="string" required>
  Detailed property description
</ParamField>

<ParamField body="direccion" type="string" required>
  Street address
</ParamField>

<ParamField body="tipo" type="string" required>
  Property type (e.g., "Casa", "Departamento", "Local", "Terreno")
</ParamField>

<ParamField body="renta_mensual" type="number" required>
  Monthly rent amount in MXN (must be >= 0)
</ParamField>

<ParamField body="deposito" type="number" optional>
  Security deposit amount in MXN (must be >= 0)
</ParamField>

<ParamField body="habitaciones" type="integer" required>
  Number of bedrooms (must be >= 0)
</ParamField>

<ParamField body="banos" type="integer" required>
  Number of bathrooms (must be >= 0)
</ParamField>

<ParamField body="metros" type="number" required>
  Property size in square meters (must be >= 0)
</ParamField>

<ParamField body="latitud" type="number" optional>
  Latitude coordinate for map display (decimal format, up to 8 decimal places)
</ParamField>

<ParamField body="longitud" type="number" optional>
  Longitude coordinate for map display (decimal format, up to 8 decimal places)
</ParamField>

<ParamField body="imagenes" type="array" optional>
  Array of image files to upload
  - Accepted formats: jpg, jpeg, png, gif, svg, webp
  - Max file size: 5MB per image
  - First image becomes the main property image (`imagen_portada`)
</ParamField>

## Implementation Details

**Source:** `app/Http/Controllers/Api/InmuebleController.php:54`

### Automatic Field Assignment

The following fields are automatically set:
- `propietario_id`: Set to the authenticated user's ID
- `ciudad`: Set to "Ocosingo"
- `estado`: Set to "Chiapas"
- `codigo_postal`: Set to "29950"
- `estatus`: Set to "disponible"

### Image Upload Process

1. Images are stored in the `public/inmuebles` directory
2. Each image creates a record in the `imagenes_inmuebles` table
3. The first uploaded image is set as the main property image (`imagen` field)
4. Image paths are stored with the `storage/` prefix
5. The URL is automatically resolved when retrieving the property

### Database Schema

**Table:** `inmuebles`

**Migration:** `database/migrations/2026_01_25_185535_create_inmuebles_table.php`

**Related Tables:**
- `imagenes_inmuebles`: Stores property images
- `usuarios`: References property owner

## Notes

- User must be authenticated and have property owner privileges
- All uploaded images are validated for format and size
- Images are stored in Laravel's public storage disk
- Coordinates (latitud/longitud) should be in decimal degrees format
- The property is immediately available for public listing after creation
