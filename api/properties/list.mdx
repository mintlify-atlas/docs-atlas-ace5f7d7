## Endpoint

```
GET /api/inmuebles/public-list
```

## Description

Returns a paginated list of all properties with `disponible` (available) status. This endpoint is public and does not require authentication.

## Query Parameters

<ParamField query="tipo" type="string" optional>
  Filter properties by type (e.g., "Casa", "Departamento", "Local")
</ParamField>

<ParamField query="ciudad" type="string" optional>
  Filter properties by city name (partial match supported)
</ParamField>

<ParamField query="page" type="integer" default="1" optional>
  Page number for pagination
</ParamField>

<ParamField query="per_page" type="integer" default="15" optional>
  Number of results per page
</ParamField>

## Request Example

```bash cURL
curl -X GET "https://api.arrendaoco.com/api/inmuebles/public-list?tipo=Casa&ciudad=Ocosingo&page=1" \
  -H "Accept: application/json"
```


## Implementation Details

**Source:** `app/Http/Controllers/Api/InmuebleController.php:21`

- Only returns properties with `estatus = 'disponible'`
- Results are ordered by most recent first (`latest()`)
- Eager loads related data: `propietario`, `imagenes`, `resenas.usuario`
- Default pagination: 15 items per page

## Notes

- This endpoint is publicly accessible (no authentication required)
- Filters are case-insensitive and support partial matching for `ciudad`
- The `tipo` filter requires an exact match
- All monetary values are in Mexican Pesos (MXN)
