## Endpoint

```
DELETE /api/inmuebles/{id}
```

<Warning>
  **Authentication Required**
  
  This endpoint requires authentication. Only the property owner or an admin can delete a property.
</Warning>

## Description

Permanently deletes a property listing from the system. This action cannot be undone.

## Authorization

The request is authorized if:
- The authenticated user is the property owner (`propietario_id` matches user ID), OR
- The authenticated user has admin privileges

**Authorization Policy:** `app/Policies/InmueblePolicy.php`

## Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the property to delete
</ParamField>

## Implementation Details

**Source:** `app/Http/Controllers/Api/InmuebleController.php:156`

### Cascade Deletion

When a property is deleted, the following related records are also automatically deleted:

**Database Schema:** `database/migrations/2026_01_25_185535_create_inmuebles_table.php`

1. **Property Images** (`imagenes_inmuebles` table)
   - Deleted via cascade constraint: `onDelete('cascade')`
   - Physical image files remain in storage (manual cleanup may be needed)

2. **Property Reviews** (`resenas` table) 
   - Deleted via model relationship

3. **Favorites** (`favoritos` table)
   - Deleted via model relationship

**Note:** Contracts (`contratos`) may prevent deletion depending on business logic implementation. Check for active contracts before deletion.

## Best Practices

<Warning>
  **Check for Active Contracts**
  
  Before deleting a property, ensure there are no active rental contracts associated with it. Deleting a property with active contracts may cause data integrity issues.
</Warning>

### Recommended Pre-Deletion Checks

1. **Verify No Active Contracts**
   ```php
   $activeContracts = $inmueble->contratos()
       ->whereIn('estatus', ['activo', 'pendiente'])
       ->count();
   
   if ($activeContracts > 0) {
       return response()->json([
           'message' => 'Cannot delete property with active contracts'
       ], 422);
   }
   ```

2. **Consider Soft Deletion**
   Instead of permanent deletion, consider setting `estatus` to `inactivo`:
   ```php
   $inmueble->update(['estatus' => 'inactivo']);
   ```

3. **Clean Up Images**
   If you want to delete physical image files:
   ```php
   foreach ($inmueble->imagenes as $imagen) {
       Storage::disk('public')->delete(str_replace('storage/', '', $imagen->ruta_imagen));
   }
   ```

## Alternative: Deactivate Instead of Delete

For properties with rental history, consider deactivating instead of deleting:

```bash
curl -X PUT "https://api.arrendaoco.com/api/inmuebles/42" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"estatus": "inactivo"}'
```

Benefits:
- Preserves rental history
- Maintains data integrity
- Can be reactivated later
- Prevents accidental data loss

## Notes

- This is a permanent deletion (hard delete, not soft delete)
- Related images in the `imagenes_inmuebles` table are automatically deleted
- Physical image files in storage are NOT automatically deleted
- Only the property owner or admin can delete a property
- Consider implementing additional business logic to prevent deletion of properties with active contracts
