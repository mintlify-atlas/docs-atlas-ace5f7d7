## Overview

This endpoint returns a paginated list of properties that the authenticated user has marked as favorites. Properties are returned in reverse chronological order (most recently added first) with complete property details.

## Authentication

<Warning>
  This endpoint requires authentication using Laravel Sanctum.
  
  Include the authentication token in the request:
  ```
  Authorization: Bearer {token}
  ```
</Warning>

## Request

### Query Parameters

<ParamField query="page" type="integer" default="1">
  The page number for pagination
</ParamField>

### Example Request

```bash cURL
curl -X GET "https://arrendaoco.com/api/favoritos" \
  -H "Authorization: Bearer {token}" \
  -H "Accept: application/json"
```

```javascript JavaScript
const response = await fetch('https://arrendaoco.com/api/favoritos', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer ' + token,
    'Accept': 'application/json'
  }
});

const data = await response.json();
```

### Key Features

- **Pagination**: Results are paginated with 15 items per page
- **Sorting**: Favorites are ordered by creation date (newest first)
- **Full Details**: Returns complete property information including images, reviews, and owner data
- **Relationship Loading**: Automatically loads related data (owner, images, reviews)

### Query Implementation

```php
$favoritos = Auth::user()->inmueblesFavoritos()->latest()->paginate(15);
return InmuebleResource::collection($favoritos);
```
