---
title: "Login"
api: "POST /api/login"
description: "Authenticate a user and receive an access token"
---

## Overview

Authenticates a user with email and password credentials. Returns a Laravel Sanctum access token and user information upon successful authentication.

## Request

### Headers

<ParamField header="Content-Type" type="string" required>
  application/json
</ParamField>

<ParamField header="Accept" type="string" required>
  application/json
</ParamField>

### Body Parameters

<ParamField body="email" type="string" required>
  User's email address
</ParamField>

<ParamField body="password" type="string" required>
  User's password
</ParamField>

## Response

<ResponseField name="token" type="string">
  Laravel Sanctum access token for authenticating subsequent requests
</ResponseField>

<ResponseField name="usuario" type="object">
  User information object
  
  <Expandable title="usuario properties">
    <ResponseField name="id" type="integer">
      User's unique identifier
    </ResponseField>
    
    <ResponseField name="nombre" type="string">
      User's full name
    </ResponseField>
    
    <ResponseField name="email" type="string">
      User's email address
    </ResponseField>
    
    <ResponseField name="foto_perfil" type="string" nullable>
      URL to user's profile photo (null if not set)
    </ResponseField>
    
    <ResponseField name="roles" type="array">
      Array of role names assigned to the user (e.g., ["inquilino"], ["propietario"])
    </ResponseField>
    
    <ResponseField name="es_admin" type="boolean">
      Whether the user has admin privileges
    </ResponseField>
    
    <ResponseField name="estatus" type="string">
      User account status ("activo" or "inactivo")
    </ResponseField>
    
    <ResponseField name="created_at" type="string">
      Timestamp when the user account was created
    </ResponseField>
  </Expandable>
</ResponseField>

## Examples

### Request

```bash
curl -X POST https://api.arrendaoco.com/api/login \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{
    "email": "usuario@example.com",
    "password": "password123"
  }'
```

### Success Response

```json
{
  "token": "1|abcdefghijklmnopqrstuvwxyz1234567890",
  "usuario": {
    "id": 1,
    "nombre": "Juan Pérez",
    "email": "usuario@example.com",
    "foto_perfil": "https://api.arrendaoco.com/storage/profiles/avatar.jpg",
    "roles": ["inquilino"],
    "es_admin": false,
    "estatus": "activo",
    "created_at": "2026-01-25 10:30:00"
  }
}
```

## Error Responses

### 401 Unauthorized - Invalid Credentials

Returned when the email/password combination is incorrect.

```json
{
  "message": "Credenciales inválidas"
}
```

### 422 Unprocessable Entity - Validation Error

Returned when required fields are missing or invalid.

```json
{
  "message": "The email field is required. (and 1 more error)",
  "errors": {
    "email": [
      "The email field is required."
    ],
    "password": [
      "The password field is required."
    ]
  }
}
```

## Using the Token

Include the returned token in the `Authorization` header for all protected endpoints:

```bash
Authorization: Bearer 1|abcdefghijklmnopqrstuvwxyz1234567890
```
