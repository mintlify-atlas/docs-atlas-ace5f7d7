## Overview

Returns a complete financial statement for a rental contract, including all payment records, totals, and balance information. Both landlords and tenants can access this endpoint for contracts they're involved in.

<ParamField path="id" type="integer" required>

## Authorization

Access is restricted to:
- The property owner (`propietario_id`)
- OR the tenant (`inquilino_id`)

```php
if (
    $contrato->propietario_id !== $request->user()->id &&
    $contrato->inquilino_id !== $request->user()->id
) {
    abort(403, 'No autorizado');
}
```

## Payment Status Definitions

- **pendiente**: Payment is due but not yet overdue
- **pagado**: Payment has been completed
- **vencido**: Payment is overdue (typically set by a scheduled job)

## Calculation Logic

### Total Paid

```php
$pagos->where('estatus', 'pagado')->sum('monto')
```

Sums the `monto` field for all payments with status `pagado`.

### Total Pending

```php
$pagos->whereIn('estatus', ['pendiente', 'vencido'])->sum('monto')
```

Sums amounts for both pending and overdue payments.

### Overdue Count

```php
$pagos->where('estatus', 'vencido')->count()
```

Counts number of payments with `vencido` status.

## PDF Export

<Info>
  A PDF version of the statement is available via a separate endpoint that generates and emails the statement.
</Info>

While not implemented in the current controller code, the routes file suggests PDF export functionality:

```bash
GET /api/contratos/{id}/estado-cuenta/excel
```

This endpoint (named "excel" but actually generates PDFs based on the controller) will:

1. Generate a PDF using the `pdf.estado_cuenta` view
2. Save it to `storage/estados_cuenta/contrato_{id}/estado_cuenta_{year}_{month}.pdf`
3. Create or update an `EstadoCuenta` record
4. Email the PDF to the tenant
5. Return the PDF as a download

### PDF Generation Details

The PDF export uses:
- **Library**: `barryvdh/laravel-dompdf`
- **View Template**: `resources/views/pdf/estado_cuenta.blade.php`
- **Paper Size**: Letter
- **Storage Location**: `storage/app/estados_cuenta/contrato_{id}/`

### Historical Statements

PDF statements are saved with unique year/month combinations:

```php
EstadoCuenta::updateOrCreate(
    [
        'contrato_id' => $contrato->id,
        'mes'  => $fecha->month,
        'anio' => $fecha->year,
    ],
    [
        'ruta_pdf'     => $pathRelativo,
        'generado_por' => $request->user()->id,
    ]
);
```

This creates a historical archive of monthly statements.

## Database Schema

### estados_cuenta Table

```php
Schema::create('estados_cuenta', function (Blueprint $table) {
    $table->id();
    $table->foreignId('contrato_id')->constrained('contratos')->cascadeOnDelete();
    $table->integer('mes');
    $table->integer('anio');
    $table->string('ruta_pdf');
    $table->foreignId('generado_por')->constrained('usuarios');
    $table->timestamps();
    
    $table->unique(['contrato_id', 'mes', 'anio']);
});
```

The unique constraint ensures only one statement per contract per month.

## Use Cases

### Tenant Checking Balance

```bash
GET /api/contratos/15/estado-cuenta
Authorization: Bearer {tenant_token}
```

Tenant views their payment history and outstanding balance.

### Landlord Reviewing Income

```bash
GET /api/contratos/15/estado-cuenta
Authorization: Bearer {landlord_token}
```

Landlord tracks received payments and identifies overdue amounts.

### Pre-Cancellation Audit

Before cancelling a contract, verify all payments are settled:

```bash
GET /api/contratos/15/estado-cuenta
```

Check that `resumen.total_pendiente` is 0 before calling the cancel endpoint.
