## Overview

Initiates a rental contract between a property owner and a tenant. This endpoint automatically:
- Creates the contract record
- Updates the property status to `rentado` (rented)
- Validates ownership and availability

<Warning>
  Only the property owner can create rental contracts. The property must have status `disponible` (available).
</Warning>

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

## Path Parameters

<ParamField path="id" type="integer" required>
  The property ID to rent out
</ParamField>

## Request Body

<ParamField body="inquilino_id" type="integer" required>
  The tenant's user ID. Must be a valid user and cannot be the property owner.
</ParamField>

<ParamField body="fecha_inicio" type="date" required>
  Contract start date in format `YYYY-MM-DD`
</ParamField>

<ParamField body="fecha_fin" type="date">
  Contract end date in format `YYYY-MM-DD`. Must be after `fecha_inicio`. Leave null for indefinite contracts.
</ParamField>

<ParamField body="renta_mensual" type="decimal" required>
  Monthly rent amount. Must be >= 0.
</ParamField>

<ParamField body="deposito" type="decimal">
  Security deposit amount. Must be >= 0 if provided.
</ParamField>

## Validation Rules

### Ownership Validation

```php
if ($inmueble->propietario_id !== $request->user()->id) {
    abort(403, 'Solo el propietario puede rentar este inmueble');
}
```

Only the property owner can create rental contracts.

### Availability Check

```php
if ($inmueble->estatus !== 'disponible') {
    abort(422, 'El inmueble no está disponible');
}
```

Property must have status `disponible` to be rented.

### Self-Rental Prevention

```php
if ($data['inquilino_id'] == $request->user()->id) {
    abort(422, 'El propietario no puede ser el inquilino');
}
```

Property owners cannot rent to themselves.

## Database Transaction

The contract creation is wrapped in a database transaction to ensure atomicity:

1. Create contract record with status `activo`
2. Update property status to `rentado`

If either operation fails, both are rolled back.

## Next Steps

After creating a contract, you typically want to:

1. **Generate payment schedule** using [POST /api/contratos/{id}/pagos/generar](/api/payments/generate)
2. **View contract details** using [GET /api/contratos](/api/contracts/list)
3. **Check payment status** using [GET /api/contratos/{id}/estado-cuenta](/api/contracts/statement)

## Payment Generation

<Info>
  Contracts do not automatically generate payment records. Use the [Generate Payments](/api/payments/generate) endpoint to create a payment schedule.
</Info>

Example:

```bash
POST /api/contratos/15/pagos/generar
{
  "meses": 12
}
```

This creates 12 monthly payment records starting from the contract's `fecha_inicio`.
