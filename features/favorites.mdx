## Overview

ArrendaOco's favorites system allows users to bookmark properties they're interested in, add personal notes, and organize their rental search.

## Favorites Model

### Database Schema

From migration `2026_02_04_172236_create_favoritos_table.php`:

```php
Schema::create('favoritos', function (Blueprint $table) {
    $table->id();
    $table->foreignId('usuario_id')
        ->constrained('usuarios')
        ->onDelete('cascade');
    $table->foreignId('inmueble_id')
        ->constrained('inmuebles')
        ->onDelete('cascade');
    $table->text('nota')->nullable();
    $table->timestamps();
    
    $table->unique(['usuario_id', 'inmueble_id']);
});
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `usuario_id` | foreign key | User who saved the favorite |
| `inmueble_id` | foreign key | Property saved as favorite |
| `nota` | text | Personal note about the property |
| `created_at` | timestamp | When favorite was added |

<Note>
The unique constraint `(usuario_id, inmueble_id)` ensures a user can only favorite a property once.
</Note>

## Adding/Removing Favorites

### Toggle Favorite

The system uses a toggle mechanism - clicking favorite either adds or removes the property.

**Web Route**: `POST /favoritos/{inmueble}/toggle`

**API Route**: `POST /api/favoritos/{inmueble}/toggle`

**Access**: Authenticated users only

**Implementation** (app/Http/Controllers/FavoritoController.php:24-54):

```php
public function toggle(Inmueble $inmueble)
{
    $usuario = Auth::user();
    
    $existe = Favorito::where('usuario_id', $usuario->id)
                      ->where('inmueble_id', $inmueble->id)
                      ->first();
    
    if ($existe) {
        // Remove from favorites
        $existe->delete();
        $mensaje = 'Eliminado de favoritos';
        $agregado = false;
    } else {
        // Add to favorites
        Favorito::create([
            'usuario_id' => $usuario->id,
            'inmueble_id' => $inmueble->id
        ]);
        $mensaje = 'Agregado a favoritos';
        $agregado = true;
    }
    
    if (request()->ajax() || request()->wantsJson()) {
        return response()->json([
            'success' => true,
            'mensaje' => $mensaje,
            'agregado' => $agregado
        ]);
    }
    
    return back();
}
```

**API Response Example**:

```json
{
  "success": true,
  "mensaje": "Agregado a favoritos",
  "agregado": true
}
```

### AJAX Implementation

Favorite buttons typically use AJAX for smooth UX:

```javascript
function toggleFavorito(inmuebleId) {
    fetch(`/favoritos/${inmuebleId}/toggle`, {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json',
            'X-CSRF-TOKEN': document.querySelector('meta[name="csrf-token"]').content
        }
    })
    .then(response => response.json())
    .then(data => {
        if (data.success) {
            // Update button icon
            const btn = document.getElementById(`fav-${inmuebleId}`);
            btn.innerHTML = data.agregado ? '❤️' : '🤍';
            
            // Show toast notification
            alert(data.mensaje);
        }
    });
}
```

## Adding Notes to Favorites

Users can add personal notes to remember details about properties.

**Web Route**: `PUT /favoritos/{inmueble}`

**API Route**: `PUT /api/favoritos/{inmueble}`

**Request Example**:

```json
{
  "nota": "Visitar el sábado a las 10am. Preguntar sobre servicios incluidos."
}
```

**Implementation** (app/Http/Controllers/FavoritoController.php:59-74):

```php
public function update(Request $request, Inmueble $inmueble)
{
    $request->validate([
        'nota' => 'nullable|string|max:500'
    ]);
    
    $favorito = Favorito::where('usuario_id', Auth::id())
                        ->where('inmueble_id', $inmueble->id)
                        ->firstOrFail();
    
    $favorito->update([
        'nota' => $request->nota
    ]);
    
    return back()->with('success', 'Nota actualizada correctamente');
}
```

<Tip>
Notes are limited to 500 characters and are completely private - only visible to the user who created them.
</Tip>

## User Model Relationship

Define the relationship in the `Usuario` model:

```php
class Usuario extends Authenticatable
{
    /**
     * Favoritos relationship (through pivot table)
     */
    public function inmueblesFavoritos()
    {
        return $this->belongsToMany(
            Inmueble::class,
            'favoritos',
            'usuario_id',
            'inmueble_id'
        )->withPivot('nota', 'created_at')
          ->withTimestamps();
    }
    
    /**
     * Direct favoritos records
     */
    public function favoritos()
    {
        return $this->hasMany(Favorito::class, 'usuario_id');
    }
}
```

## Inmueble Model Relationship

```php
class Inmueble extends Model
{
    public function usuariosFavoritos()
    {
        return $this->belongsToMany(
            Usuario::class,
            'favoritos',
            'inmueble_id',
            'usuario_id'
        );
    }
}
```

## Use Cases

### UC-26: Alta de Favorito

**Actor**: Tenant

1. User views property details

3. System creates `Favorito` record

2. Clicks "Remove from Favorites" button

### UC-28: Edición de Favorito

**Actor**: Tenant

**Flow**:
1. User views favorites list
2. Adds/edits note for a property
3. System updates `nota` field
4. Note is saved and displayed

**Actor**: Tenant

**Flow**:
1. User navigates to `/favoritos`
2. System displays all favorited properties
3. User can see notes and manage favorites

## Cascade Deletion

<Warning>
When a property is deleted, all associated favorites are automatically removed via `onDelete('cascade')`.
When a user account is deleted, all their favorites are removed.
</Warning>

## Best Practices

### For Users

1. **Use notes**: Add reminders like appointment times, questions to ask
2. **Regular review**: Check favorites before they're rented by others
3. **Compare options**: Keep multiple favorites to compare features and prices

### For Developers

1. **AJAX for better UX**: Toggle favorites without page reload
2. **Visual feedback**: Clearly show favorite status with icons/colors
3. **Pagination**: Use pagination for users with many favorites
4. **Notification**: Notify users if a favorited property becomes unavailable
